接口

概念：接口用于表述类具有什么功能，但是并不给出具体的实现。类要遵从接口描述的统一规则进行定义，所以，接口是对外提供的一组规则，标准。

  • 接口是比抽象类更抽象的存在
  • 定义接口用关键字interface
  • 类和接口是实现的关系 class 类名 implements 接口名

接口的特点

接口创建对象的特点

  • 接口不能实例化。 可以通过多态的方式创建对象。
  • 接口的子类有两种：
    1. 可以是普通类，必须实例化所有抽象方法。
  1. 可以是抽象类，不用实例化方法,可以全部不重写或只重写一部分方法。

接口继承关系的特点

接口与类之间的关系特点：

  • 类与类的关系：继承关系，只能单继承。
  • 接口与类的关系：实现关系，可以多层次实现。

接口与接口之间的关系：

  • 继承关系，可以多继承，格式：接口 extends 接口1，接口2，接口3…

继承和实现的区别：

  • 继承体现的是“is a”的关系，父类定义共性的内容
  • 实现体现的是“like a”的关系，接口中定义扩展功能

接口成员的特点

成员变量：

  • 接口没有成员变量，只有共有的，静态的常量。（jdk会默认变量是 public abstract final）(记得大写常量变量名)

成员方法

  • JDK7以前，公有的抽象方法： public abstract 返回值类型 方法名() {}。
  • JDK8以后，可以有默认方法和静态方法： public default 返回值类型 方法名 () {} , static 返回值类型 方法名 () {}。
  • JDK9以后，可以有私有方法： private 返回值类型 方法名 () {}。

构造方法：

  • 接口中没有构造方法，因为接口中没有变量也不需要实例化。
