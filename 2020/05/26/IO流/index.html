I/O流

异常

概述

程序出现错误

异常的分类

Exception:合理的应用程序可能捕获的问题。e.g：NullPointerException

Error:合理的应用程序不应该试图捕获的问题 e.g：StackOverFlowError

异常的处理方式

  • try catch (finally) :捕获，自己处理

特点：处理完异常之后程序会继续执行。

在try或catch里写了return的话（代码终止），想要执行代码，必须要有finally。

执行流程：

  • 先执行try（）中的内容，看是否有异常

  • 没有：直接执行finally语句的内容

  • 有：跳转到catch（）{}语句开始执行，再执行finally（）中的内容。

  • throws:抛出异常，交给调用者处理

I/O:Java中传输数据的方式

分类：

按数据流向：输入流和输出流

按操作方式：字节流(InputStream & OutputStream)和字符流(Reader & Writer)

File类

概念：文件夹，文件，一个file对象代表磁盘上的某个文件或文件夹

  • 构造方法

File(String pathname) 根据字符串形式的路径获取file对象

File(String parent, String child) 根据字符串形式的父目录以及子目录创建file对象

File(File parent, String child) 根据父目录对象，以及子字符串形式来获取file对象

  • 成员方法

creatNewFile():创建文件。如果不存在就创建返回true，如果存在就不创建返回false。

mkdir() & mkdirs():创建目录

isDirectory():判断File对象是否为文件夹(目录)

isFile():判断File对象是否为文件

exists():判断File对象是否存在

getAbsolutePath():获取绝对路径

绝对路径：以盘符开头的路径

getPath():获取相对路径

getName():获取文件名

list():获取指定目录下所有文件（夹）名称数组

listFile():获取指定目录下所有文件（夹）File数组

IO流拷贝文件核心6步

  1. 创建字符输入流对象，关联数据源文件。
  2. 创建字符输出流对象，关联目的地文件。
  3. 定义变量，记录读取到的内容。
  4. 循环读取，只要条件满足就一直读，并将读取到的内容赋值给变量。
  5. 将去读到的数据写入到目的地文件中。
  6. 释放资源。

字符缓冲流的常用成员方法
